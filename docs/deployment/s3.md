# S3

hub と service-ui の静的アセット配置先。バケットを **2 つ** 用意する。

## バケット設定

| 項目 | 値 |
|---|---|
| Region | CloudFront に近い任意のリージョン（東京なら ap-northeast-1） |
| Versioning | Enabled 推奨（誤削除リカバリ） |
| Block all public access | **すべて ON** |
| Default encryption | SSE-S3 (AES-256) または SSE-KMS |
| Object Ownership | Bucket owner enforced（ACL 無効） |
| Static website hosting | **無効**（CloudFront + OAC 経由でのみ配信するため） |

バケット名例:
- `myapp-hub-prod`
- `myapp-service-ui-prod`

## バケットポリシー（OAC 用）

CloudFront の Origin Access Control を作成すると、AWS コンソールが自動でこのテンプレートを提示する。`{{BUCKET_NAME}}` と `{{ACCOUNT_ID}}` と `{{DISTRIBUTION_ID}}` を置換:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowCloudFrontServicePrincipalReadOnly",
      "Effect": "Allow",
      "Principal": { "Service": "cloudfront.amazonaws.com" },
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::{{BUCKET_NAME}}/*",
      "Condition": {
        "StringEquals": {
          "AWS:SourceArn": "arn:aws:cloudfront::{{ACCOUNT_ID}}:distribution/{{DISTRIBUTION_ID}}"
        }
      }
    }
  ]
}
```

## ビルド & sync

```sh
# 環境変数を本番値で渡してビルド
VITE_SERVICE_UI_URL=https://service-ui.example.com/ \
VITE_SERVICE_UI_ORIGIN=https://service-ui.example.com \
pnpm --filter hub build

VITE_ALLOWED_PARENT_ORIGINS=https://hub.example.com \
VITE_API_BASE=https://api.example.com \
VITE_DEFAULT_SERVICE_ID=service-001 \
pnpm --filter service-ui build
```

成果物は `apps/hub/dist/` と `apps/service-ui/dist/`。

### キャッシュヘッダを分けて sync

ハッシュ付きアセットは長期キャッシュ、`index.html` は no-cache。

```sh
# hub
aws s3 sync apps/hub/dist s3://myapp-hub-prod \
  --delete \
  --exclude "index.html" \
  --cache-control "public,max-age=31536000,immutable"
aws s3 cp apps/hub/dist/index.html s3://myapp-hub-prod/index.html \
  --cache-control "no-cache,no-store,must-revalidate"

# service-ui
aws s3 sync apps/service-ui/dist s3://myapp-service-ui-prod \
  --delete \
  --exclude "index.html" \
  --cache-control "public,max-age=31536000,immutable"
aws s3 cp apps/service-ui/dist/index.html s3://myapp-service-ui-prod/index.html \
  --cache-control "no-cache,no-store,must-revalidate"
```

### CloudFront invalidation

`index.html` だけ無効化すれば足りる（他はファイル名がハッシュで変わる）。

```sh
aws cloudfront create-invalidation \
  --distribution-id $HUB_DIST_ID \
  --paths "/index.html" "/"

aws cloudfront create-invalidation \
  --distribution-id $SERVICE_UI_DIST_ID \
  --paths "/index.html" "/"
```

## Vite の env と機密の扱い

`VITE_*` で始まる環境変数は **ビルド時にバンドルへ展開される**。生成 JS を grep すれば値が読める。

- 入れていい: 公開 URL（`VITE_SERVICE_UI_URL` 等）、許可 origin、公開 ID
- 入れてはいけない: API キー、署名鍵、シークレット
- `VITE_DEFAULT_API_KEY` のような学習用の値も、本番 build からは抜く（標準入力を空にしてビルド、もしくはビルド側のコードを削除）

シークレットはバックエンド側で扱い、フロントは Cookie / Authorization ヘッダ経由で受け取る設計に寄せる。
